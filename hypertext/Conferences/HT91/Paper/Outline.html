HT91 paper: Outline - Draft

This is an outline for a paper for possible presentation to the HT91 conference . See also: The paper itself .

The aim of the paper is to present the WWW as a set of solutions to some demonstrated problems. The WWW solutions differ from most of those presented at HT90, primarily in the following points:

  • The link information is stored in or with the document, not in a central repository.
  • There is a blending of index search and link jumping (ref Microcosm) in that an index search result is normal hypertext.
  • There is data format negociation between client and server, to allow continuous development of new representations.
  • More intelligence is vested in the server, in its ability to do searches and format conversions, reducing network load, and allowing different indexing schemes to be used on different machines.
