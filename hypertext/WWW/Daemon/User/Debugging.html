Debugging the daemon

Suppose you think you have installed a W3 server but it doesn't work. That is, you have followed the installation instructions and the test at the end fails. Here we assume you have used port 80. If you have a situation not handled by this problem-solving guide, please mail me.

Type

www http://myhost.domain:80/ What happens?
  • "Cannot connect to information server" message .
  • An empty document is displayed
  • A document containing the words "Document address invalid or access not authorised" is displayed.
  • A document is displayed, but not what you wanted the server to give in response to that document name (/)
