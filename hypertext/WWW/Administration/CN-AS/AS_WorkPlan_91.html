WWW Work Plan for AS group 1991

This is a contribution from the WWW project to the AS group work plan for 1991. (See the previous AS group work plan for 1990 ).

Note that this phase requires more manpower than we currently have. we will require a replacement for Nicola Pellow who leaves at the end of the summer, plus one more programmer.

1991 sees the end of the initial phase of the WWW project. The existence of two browsers in extremely different environments, two servers in extremely different environments, several large indexes and a web of many hypertext documents has proven the viability of the system. The next phase of the project implements its penetration onto all platforms, and extension to groups outside the CERN site.

General

  • Implement format negociation feature of the protocol to allow exchange and conversion of differing data formats.
  • Implement document caching for documents it takes a long time to retrieve or convert.

Browsers

Line Mode Browser

  • Implement reverse scrolling, and various requested features
  • Increase penetration to all CERN machine types
  • Integrate Internet news access mechanism
  • Public release to academic/H.E.P. community.

NeXTStep Browser

  • Integrate Anonymous-FTP access mechanism
  • Prepare for release to the academic NeXTStep community for comment

X Browser

  • Develop. Manpower requirement exceeds that of phase 0, so this item cannot be scheduled for 1991 without additional resources. An IBM/PC browser would also be considered if the manpower were increased.

Mac Browser

  • Develop browser for Macintosh.

Servers and Data Provision

  • WHO data from CERNVM (see DE section work plan)
  • Develop and distribute generic unix server for man pages etc, including indexing using public domain code;
  • VMS server for VMS/HELP files
  • Set up librarian services for catalogues of available data, and overview documents.

External contacts

  • Other information service software projects, e.g. W.A.I.S. ) and hypertext projects (e.g. DynaText ).
  • Awareness of, distribution to, and feedback from H.E.P. groups: experiments, other labs.
  • Collaboration with other information providers, such as research networks user support (CONCISE etc), Archie, existing W.A.I.S. servers, etc.
