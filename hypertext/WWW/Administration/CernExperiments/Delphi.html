WWW for Delphi

Contacts have been made with Mark Donszelmann about possible uses of W3 for the Delphi online slow controls team. Three projects were launched after discussion with JFG on Feb. 12, 1992 :

Bookreader documentation

Currently, most of Delphi documentation is managed with DEC's Bookreader system, which involves writing SDML source files and processing them to obtain screen (DECWindows) versions and printer (PostScript) versions, all stored separately. Using HTML instead would allow broader access from any machines and terminals, and would only require to store one version, i.e. the HTML source.
