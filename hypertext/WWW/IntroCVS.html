  • more experiments with CVS give good feeling (except that we did not manage to remove a file). When a developer requests an update for a file of which he has a copy "A", CVS looks at the original version "O" that he checked out AND at the latest version "L" (as checked in by the other developer). If "L" differs from "O" in a part where "O" and "A" are the same, then that difference is propagated into "A". If "L" differs from "O" in a part where "A" also differs from "O". then that is a conflict. It is marked in "A" and a backup of the original "A" (before the update" is put in ".#A".
